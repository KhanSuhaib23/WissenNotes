**Java Intro** *Java Features* * Object Oriented + Functional Programming (since Java 8) * Statically typed (variable have a given type that doesn't change throughtout its scope) * Portable (compile once to byte code then run on anything with a JVM) * Concurrency & Parallelism (extensive support for concurrent programing) *Default Threads in Java* * Java has by default 1 main thread and 21 background threads that JVM creates * The background thread handles things like garbage collection. *Code Compilation and Running Guidelines* * Don't use default package for anything * When compiling using command line use: * javac -d [output-directory] [java-file] * It outputs all the class files in the specified output-directory * Then run using: * java [package-name].[filename] *General Structure of a Java Application* * app * bin [all the class files] * lib [all the external jar files that are used in the project] * res [resources] * src [source code] * package 1 * package 1.1 * package 1.2 * package 2 * package 2.1 * package 2.1.1 * package 2.2 !!! note Packages are used to organize code in modules and to avoid name collisions !!! note Use atleast 3 levels of package name !!! note Generally the three levels are given by
[company-type].[company-name].[module-name]
They could then be followed by submodules
e.g. com.wissen.db (com = commercial) *Package Syntax* - package root; // inside package root - package root.sub1.sub2; // inside package sub2 which is inside sub1 which is inside root *Import Syntax* - import pkg1.pkg2.ClassName; - import pkg1.pkg2.*; // All classes, not recursive *Java File Structure* package declarations; [1] import statements; [n] {class , interface , enum , @interface (annotation) } [name] {} [n] public {class | interface | enum | @interface (annotation) } [name] {} [1] *Java Objects* * State (data) * Behaviour (methods) * Identity (address) *Java Class* * Template to create a java object * Describes object with State and Behaviour *Java Class Structure* ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~ java public class Employee { // I. state ==> variables // a. class/static variables public final static String COMP_NAME = "Wissen"; public static String tnrName; // b. instance/object variables public int id; public String name; // II. constructor public Employee() { // TODO Auto-generated constructor stub } // III.behavior ==> methods // a. class/static methods public static void teach() { System.out.println("Employee :: teach()"); } // b. instance/object methods public void listenAndWork() { System.out.println(this.name + " listenAndWork()"); } // IV. nested types [public , protected , private ] [static] class InnerClass {} // V. code blocks static { } } ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~ *Object Oriented Concepts* * Abstraction / Interface / Front-End / API * Easy to use * Loose coupling between dependent and dependency * Encapsulation / Implementation / Backend * The actual implementation behind the interface * Inheritance * Reusability * Generalization / Specialization * Polymorphism * Many forms of the same thing (Cat, Dog, etc. are many forms of Animal)
